---
description: How to finalize any feature, API, UI, or database architecture task
---

1. Before notifying the user of task completion, you MUST open `manifest.md`.
2. Document new security features, changes to UI, platform capabilities, architectural changes, formula changes, API routes, Database Schema changes, new application features and any other changes.
3. If any part of the above is deprecated or decommissioned, document the change as well.
4. Update the `USER_GUIDE.md` to reflect any changes made.
5. ONLY after `manifest.md` and `USER_GUIDE.md` are completely updated may you mark the task as complete and notify the user.
